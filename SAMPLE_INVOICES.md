# Adding Invoices to the Accuracy Suite

The accuracy suite validates extraction quality against real PDFs with known expected values. It calls the configured LLM API and is intended for **local runs only** (not CI).

## Directory Structure

```
tests/accuracy/fixtures/
├── digital/
│   ├── acme_invoice_001.pdf
│   └── acme_invoice_001.expected.json
├── scanned/
│   └── ...
└── edge_cases/
    └── ...
```

## Adding a New Fixture

1. Place your PDF in the appropriate subdirectory (`digital/`, `scanned/`, or `edge_cases/`).

2. Create a matching `.expected.json` file with the same base name:

```json
{
  "invoice_number": "INV-123",
  "vendor_name": "Vendor Name",
  "invoice_date": "2025-01-15",
  "due_date": "2025-02-15",
  "total_amount": 1234.56,
  "tax_amount": 100.00,
  "currency": "USD",
  "min_confidence": {
    "invoice_number": 0.8,
    "vendor_name": 0.8,
    "invoice_date": 0.8,
    "due_date": 0.6,
    "total_amount": 0.8,
    "tax_amount": 0.6
  }
}
```

3. Add a test case in `test_real_invoices.py` following the `test_acme_invoice_001` pattern.

4. Run locally:

```bash
cd backend
OPENAI_API_KEY=sk-your-key pytest tests/accuracy/ -v -m accuracy --tb=short
# Or with Gemini:
LLM_PROVIDER=gemini GEMINI_API_KEY=your-key pytest tests/accuracy/ -v -m accuracy --tb=short
```

## Tips

- Use **digital PDFs** generated by software (not scans) for the most reliable baseline tests.
- For scanned fixtures, expect lower confidence thresholds.
- Include edge cases: multi-page, German number format, Indian lakhs, missing tax, ambiguous dates.
- Keep expected values verified by manual review of the PDF.

## Cost

Each accuracy test run costs approximately $0.002–0.05 per PDF depending on digital vs scanned routing.
